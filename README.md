# MenuApiProject

Бэкенд-сервис с **UI-меню через Swagger**. Вместо веб-интерфейса – Swagger UI как интерактивная панель управления для запуска бэкенд-логики.

## 🎯 Назначение

Дать разработчику UI-подобный опыт без фронтенда. Быстро переключаться между блоками функциональности: работа с базой данных, логические задачи (алгоритмы, валидация, вычисления), отправка сообщений в Kafka (в разработке). Служить демо-стендом для микросервисной архитектуры.

## 🧩 Меню (группы эндпоинтов в Swagger)

| Группа | Назначение |
|--------|-------------|
| DB Operations | CRUD, запросы к базе данных |
| Logic Tasks | Алгоритмические задачи (палиндром, факториал, сортировка и т.п.) |
| Kafka Publisher | Отправка событий в топик (готовится) |

## 🚀 Быстрый старт

Клонировать репозиторий: `git clone <your-repo-url> cd MenuApiProject`. Собрать проект: `mvn clean install`. Запустить приложение: `mvn spring-boot:run`. Открыть меню (Swagger UI): [http://localhost:8080/swagger-ui/index.html](http://localhost:8080/swagger-ui/index.html)

## 🕹️ Как пользоваться (имитация UI)

Выбери группу (например, DB Operations или Logic Tasks). Разверни нужный эндпоинт. Нажми Try it out → введи параметры → Execute. Получи ответ от сервера – это результат выбранной логики. Зачастую ответ будте в самом бизнес сервисе (лень прописывать логику с 2х сторон)

## 📁 Структура проекта

src/main/java/com/example/ – api/ (сгенерированные интерфейсы контроллеров), dto/ (DTO генерируются из YAML), invoker/ (вспомогательные классы), service/ (реализация логики), config/ (конфигурации). src/main/resources/ – yaml/openapi-api.yaml (спецификация API), application.properties (порт, настройки БД, Kafka).

## 🧠 Примеры логических задач (блок Logic Tasks)

Проверка числа на простоту / палиндромность. Вычисление факториала, чисел Фибоначчи. Сортировка массива (пузырьком, быстрая). Валидация строк (email, телефон, дата). Имитация состояний конечного автомата.

## 🗄️ Работа с БД (блок DB Operations)

Используется библиотека dto-lib, которая содержит JPA-сущности (схемы nsi/public), DAO для работы с БД (PublicDao, NsiDao), автоматическое заполнение справочников при старте. Поддерживаемые операции: создание/редактирование/удаление пользователей, добавление мест работы (профессия + компания + зарплата), покупки товаров (списание со счёта, уменьшение остатков), история транзакций и движения товаров.

## 🔮 Планы (Kafka)

Добавить продюсера как отдельный пункт меню. Сообщения будут уходить в топик. Отдельный сервис-консюмер будет их обрабатывать. Имитация асинхронной обработки задач.
